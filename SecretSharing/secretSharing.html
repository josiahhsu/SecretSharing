Secret Sharing Demo

This is a demo of secret sharing, which is a security scheme where a message is protected by being split into shares. In this scheme, n shares are generated such that no individual share reveals any information about the original message. Once these n shares are generated, the message can only be reconstructed if k shares are used, where k is less than or equal to n. This is known as the (k, n) threshold scheme, where k is the threshold for reconstruction and n is the number of shares actually generated. Secret sharing in this form is credited to Adi Shamir and George Robert Blakley, both of whom developed this scheme independently.

In this example, the splitting and reconstruction algorithms use a simple XOR scheme. For splitting, the message is first encoded as an array of charcodes. n-1 shares are then generated by creating random pads the same length as this array. The nth share is then generated by XORing the original charcode array with all the other randomly generated shares. Recombination simply XORs the n shares together to get the original array of charcodes, which is then decoded to reconstruct the original message. The mechanics of the XOR scheme dictate that the number of shares needed for reconstruction is exactly equal to the number of shares generated, meaning k = n. More sophisticated secret sharing schemes use methods like interpolation for splitting and reconstructing shares, which allows k to be less than n.

This demo lets you choose what value of n to use. You can then either enter a message to be split or enter shares to be reconstructed. Click "Split" to split the message into n shares, and click "Combine" to combine the n shares into the message.
